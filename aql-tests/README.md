# AQL Tests Runner

> Language / 언어 / 语言: **English** (primary) · [한국어](#한국어) · [中文](#中文)

A test runner for processing activedb query files with automatic GitHub issue creation for errors.

## Features

- Process individual query files or all files (1-100) in parallel
- Automatic GitHub issue creation for compilation and cargo check errors
- Duplicate issue prevention using error hashing
- Clean temporary directory management

## Usage

### Basic Usage

```bash
# Process all files in parallel
cargo run --bin test

# Process a specific file
cargo run --bin test -- 42
```

### GitHub Integration

To enable automatic GitHub issue creation, set the following environment variables:

```bash
export GITHUB_TOKEN="your_personal_access_token"
export GITHUB_OWNER="HelixDB"           # Optional, defaults to "HelixDB"
export GITHUB_REPO="helix-db"           # Optional, defaults to "helix-db"
```

#### Creating a GitHub Personal Access Token

1. Go to GitHub Settings → Developer settings → Personal access tokens
2. Generate a new token with the following permissions:
   - `repo` (Full control of private repositories)
   - `issues` (Read and write access to issues)
3. Copy the token and set it as the `GITHUB_TOKEN` environment variable

### How Issue Creation Works

When an error occurs during processing:

1. **Error Hashing**: A unique hash is generated from the error type and message
2. **Duplicate Check**: The system searches for existing issues with the same error hash
3. **Issue Creation**: If no duplicate exists, a new issue is created with:
   - Descriptive title including error type and file number
   - Detailed error information in the body
   - Automatic labels: `bug`, `automated`, `aql-tests`
   - Unique error hash for duplicate prevention

### Error Types Handled

- **AQL Compilation Errors**: Issues created when `activedb compile` fails
- **Cargo Check Errors**: Issues created when `cargo check` fails

### Example Issue

```
Title: Auto-generated: AQL Compilation Error in file42

Body:
## Automatic Error Report

**Error Type:** AQL Compilation
**File:** file42
**Error Hash:** ERROR_HASH:abc123def456

### Error Details
```
[Error output details]
```

---
*This issue was automatically generated by the aql-tests runner.*
```

## Requirements

- Rust 1.70+
- `activedb` CLI tool installed
- GitHub personal access token (for issue creation)
- Internet connection (for GitHub API calls)

## Dependencies

- `tokio` - Async runtime
- `clap` - Command line argument parsing
- `anyhow` - Error handling
- `octocrab` - GitHub API client
- `sha2` - Error hashing
- `base64` - Hash encoding

## 한국어

# AQL 테스트 러너

오류 발생 시 자동으로 GitHub 이슈를 생성하는 activedb 쿼리 파일 처리 테스트 러너입니다.

### 주요 기능

- 개별 쿼리 파일 또는 전체 파일(1-100)의 병렬 처리
- 컴파일 오류 및 cargo check 오류에 대한 자동 GitHub 이슈 생성
- 에러 해싱을 통한 중복 이슈 방지
- 임시 디렉토리 자동 정리

### 사용법

```bash
# 모든 파일을 병렬로 처리
cargo run --bin test

# 특정 파일만 처리
cargo run --bin test -- 42
```

### GitHub 연동

자동 이슈 생성을 활성화하려면 다음 환경 변수를 설정하세요:

```bash
export GITHUB_TOKEN="your_personal_access_token"
export GITHUB_OWNER="HelixDB"           # 선택, 기본값 "HelixDB"
export GITHUB_REPO="helix-db"           # 선택, 기본값 "helix-db"
```

#### 이슈 생성 동작
1. **오류 해싱**: 오류 타입과 메시지로부터 고유 해시를 생성
2. **중복 검사**: 동일한 해시의 기존 이슈가 있는지 검색
3. **이슈 생성**: 중복이 없으면 타입 / 파일 번호가 포함된 제목과 상세 오류 본문으로 이슈를 생성

### 요구 사항

- Rust 1.70 이상
- `activedb` CLI 설치
- GitHub personal access token (이슈 생성용)
- 인터넷 연결 (GitHub API 호출용)

## 中文

# AQL 测试运行器

用于处理 activedb 查询文件的测试运行器,可在发生错误时自动创建 GitHub issue。

### 核心特性

- 并行处理单个查询文件或全部文件(1-100)
- 为编译错误和 cargo check 错误自动创建 GitHub issue
- 通过错误哈希避免重复创建 issue
- 自动清理临时目录

### 使用方法

```bash
# 并行处理全部文件
cargo run --bin test

# 处理特定文件
cargo run --bin test -- 42
```

### GitHub 集成

如需启用自动 issue 创建,请设置以下环境变量:

```bash
export GITHUB_TOKEN="your_personal_access_token"
export GITHUB_OWNER="HelixDB"           # 可选,默认为 "HelixDB"
export GITHUB_REPO="helix-db"           # 可选,默认为 "helix-db"
```

#### Issue 创建流程
1. **错误哈希**:根据错误类型和消息生成唯一哈希
2. **重复检查**:查询是否已存在相同哈希的 issue
3. **创建 Issue**:若无重复,则以包含类型和文件编号的标题及详细错误正文创建新 issue

### 要求

- Rust 1.70 及以上
- 已安装 `activedb` CLI
- GitHub 个人访问令牌(用于创建 issue)
- 网络连接(用于调用 GitHub API)
